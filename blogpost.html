It Started With My PI

My PI, Dr. Jolene Ramsey, had a simple request early in my PhD: "Plot your graphs in R." Fair enough — R produces beautiful, reproducible, publication-quality figures. But every time I had a new experiment, I was writing the same ggplot2 code from scratch. Adjusting axes. Moving the legend. Re-running everything when I got one more replicate. It was tedious.

"If you want to use GraphPad, you need to pay for it. In Excel you have to modify things slowly over time. They are both great tools and have done so much for scientists — but I do a lot of lysis curves and I did not want to always code things manually."

The turning point came in my introductory Experimental Design course in grad school. The most valuable lesson wasn't a statistical method or a study design framework — it was learning that R Shiny exists. The idea that you can wrap any analysis in a point-and-click interface, accessible from a browser, with no programming knowledge required — that clicked for me immediately.

I may have overdone it since then. I now try to automate anything with Shiny apps. But this one — the Lysis Curve App — is the one I use almost every day.

The Variability Display Modes

One of the most useful additions to the app is the expanded set of ways to show biological variability. Traditional tools give you error bars or nothing. This app gives you seven:

ModeWhat you seeBest for
Error Bars ±SD, SEM, or 95% CI as T-bars or line bars Standard publication figures
Shadow / Ribbon Filled band around the mean Clean, modern look; dense curves
Spaghetti Each replicate as a semi-transparent trace Showing timing differences between replicates
Quantile Bands Nested IQR (25–75%) + 95% range Honest representation of spread without assuming symmetry
Jitter Points Every raw data point scattered at each timepoint Small n; showing distribution and outliers
Combo Spaghetti traces + light CI ribbon + bold mean Maximum information; presentations
None Mean line only Exploratory viewing; many samples

How to Get & Run It

LATEST RELEASE — v2.0.0

No installation beyond R and a few packages.
⬇ Download app.R Release notes ↗

The entire app is a single R file. No server, no deployment, no subscription. Install R (free), install the packages below (one paste into the console), then run the file.

Step 1 — Install prerequisites

You need R ≥ 4.2 and ideally RStudio. Both are free.

  • Download R from CRAN
  • Download RStudio from Posit (recommended)

Step 2 — Install packages

Paste this into the R console once. Everything else is handled automatically.

# Required packages install.packages(c( "shiny", "tidyverse", "ggpubr", "scales", "ggrepel", "ggprism", "svglite", "jsonlite", "zoo", "DT" )) # Optional — Excel-paste grid in Data Entry tab (v2.0+) install.packages("rhandsontable") # Optional — PowerPoint export install.packages(c("officer", "rvg")) # Optional — animated GIF export install.packages("gifski")

Step 3 — Download and run the app

Use the download button above, or clone the repo. Then in RStudio:

  • Open the app.R file in RStudio
  • Click Run App in the toolbar — or run from the console:
shiny::runApp("path/to/app.R")

The app will open in your default browser. Keep the R console window open while using it.

Step 4 — Load your data

Your plate-reader CSV goes straight in. The app auto-detects wide format (one column per sample) or long format (a grouping column named condition, sample, etc.). Replicates in wide format: stack the time blocks vertically — the app detects where the time counter resets and assigns replicate IDs automatically.

FormatWhen to useReplicate detection
Wide (recommended) One column per sample, one row per timepoint Stack replicate blocks; time resets detected automatically
Long A grouping column named condition, sample, group, etc. Column named replicate, rep, or well

Step 5 — Customize, analyze, export

  • Pick samples, set colors/shapes/line types in the Plot tab
  • Choose a variability mode from the Variability / Error Display panel
  • Click Calculate Metrics in the Analysis tab to compute all 19 growth metrics
  • Fill in the Experiment Notes tab for a full lab record
  • Click Save Settings to export a JSON that restores everything next session
  • Download your plot in any format from the Export panel

Full data format details, all 19 metric definitions, and troubleshooting tips are in the README on GitHub.

Why Not Just Use GraphPad or Excel?

GraphPad Prism is excellent — polished, well-documented, statistically robust. Excel is everywhere and gets the job done for simple plots. Both tools have done enormous amounts of good for scientists, and I mean no disrespect to either.

But GraphPad costs money (a lot of it for a PhD student on a stipend). Excel requires manual tweaking on every new dataset, and the graphs rarely look the way you want without significant effort. And neither is reproducible in the way that code-based tools are.

R Shiny gives you the best of both worlds: a point-and-click interface (no code needed to use it) built on a reproducible, programmable foundation — R, ggplot2, and a JSON settings file that restores your exact preferences next session. And it is completely free.

Data Entry — no CSV required

The new Data tab has two ways to get data into the app without preparing a file first. Option A is a spreadsheet grid (powered by rhandsontable): set the number of rows and columns, then paste directly from Excel with Ctrl+V. Option B is a textarea — paste any comma- or tab-separated text, let the app auto-detect the separator, preview the first ten rows, and click Apply. Either way, clicking Use This Data as Active Dataset runs the same format-detection pipeline as a file upload and switches straight to the Plot tab.

Curve fitting — what the parameters mean

The Logistic model fits: OD(t) = A / (1 + e−k(t − t₀))
The Gompertz model fits: OD(t) = A · exp(−exp(−k(t − t₀)))

In both cases A is the asymptotic maximum OD (carrying capacity), k is the growth rate constant (related to μmax), and t₀ is the inflection point — where growth rate is highest for the logistic model, and roughly the time of maximum acceleration for Gompertz. R² measures how well the model fits the mean curve. Both models are fitted using R's built-in nls() (nonlinear least squares). If convergence fails — which can happen with noisy data or an unclear growth phase — the app skips that sample and reports which ones succeeded.
