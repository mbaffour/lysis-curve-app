It Started With My PI

My PI, Dr. Jolene Ramsey, had a simple request early in my PhD: "Plot your graphs in R." Fair enough — R produces beautiful, reproducible, publication-quality figures. But every time I had a new experiment, I was writing the same ggplot2 code from scratch. Adjusting axes. Moving the legend. Re-running everything when I got one more replicate. It was tedious.

"If you want to use GraphPad, you need to pay for it. In Excel you have to modify things slowly over time. They are both great tools and have done so much for scientists — but I do a lot of lysis curves and I did not want to always code things manually."

The turning point came in my introductory Experimental Design course in grad school. The most valuable lesson wasn't a statistical method or a study design framework — it was learning that R Shiny exists. The idea that you can wrap any analysis in a point-and-click interface, accessible from a browser, with no programming knowledge required — that clicked for me immediately.

I may have overdone it since then. I now try to automate anything with Shiny apps. But this one — the Lysis Curve App — is the one I use almost every day.

What the App Does

You upload a CSV from your plate reader. You pick your samples. You click a few buttons. You get a publication-ready figure with error bars, a full set of growth metrics, and statistical comparisons — all customized to your exact preferences, saved and re-loaded between sessions.

The Variability Display Modes

One of the most useful additions to the app is the expanded set of ways to show biological variability. Traditional tools give you error bars or nothing. This app gives you seven:

ModeWhat you seeBest for
Error Bars ±SD, SEM, or 95% CI as T-bars or line bars Standard publication figures
Shadow / Ribbon Filled band around the mean Clean, modern look; dense curves
Spaghetti Each replicate as a semi-transparent trace Showing timing differences between replicates
Quantile Bands Nested IQR (25–75%) + 95% range Honest representation of spread without assuming symmetry
Jitter Points Every raw data point scattered at each timepoint Small n; showing distribution and outliers
Combo Spaghetti traces + light CI ribbon + bold mean Maximum information; presentations
None Mean line only Exploratory viewing; many samples

How to Run It

The entire app is a single R file. No server, no deployment, no subscription. Install R (free), install the packages (one paste into the console), open the file.

# Step 1 — Install required packages (one-time) install.packages(c( "shiny", "tidyverse", "ggpubr", "scales", "ggrepel", "ggprism", "svglite", "jsonlite", "zoo", "DT" )) # Optional — PowerPoint export install.packages(c("officer", "rvg")) # Step 2 — Download the app file from GitHub, then run it shiny::runApp("path/to/app.R")

Full instructions, data format details, and all 19 metric definitions are in the README on GitHub.

Why Not Just Use GraphPad or Excel?

GraphPad Prism is excellent — polished, well-documented, statistically robust. Excel is everywhere and gets the job done for simple plots. Both tools have done enormous amounts of good for scientists, and I mean no disrespect to either.

But GraphPad costs money (a lot of it for a PhD student on a stipend). Excel requires manual tweaking on every new dataset, and the graphs rarely look the way you want without significant effort. And neither is reproducible in the way that code-based tools are.

R Shiny gives you the best of both worlds: a point-and-click interface (no code needed to use it) built on a reproducible, programmable foundation — R, ggplot2, and a JSON settings file that restores your exact preferences next session. And it is completely free.
